How to set up a Permanent Session (Cloud / GCP)

Cloud servers lack graphical interfaces. To use proXiPT seamlessly 24/7 on a remote server:

  1. Run proXiPT once on your Local PC and use the Login (GUI) button to sign in.
  2. This creates a session.json file in your local sessions/ folder.
  3. On your Cloud server, click the Upload JSON button below for the respective provider and select that file.
  4. proXiPT will automatically keep the session alive indefinitely from that point forward!

Vibe-Coding Integrations (Top 10)

Copy these settings into your favorite coding GUI tools to use proXiPT as a local AI endpoint.

1. Claude Code (Anthropic CLI)

1. Claude Code only speaks Anthropic API format. To bridge it to ProxiPT, install litellm:
pip install litellm
2. Start LiteLLM proxy to translate Anthropic requests to ProxiPT's OpenAI format:
litellm --model openai/auto --api_base http://localhost:8787/v1 --api_key dummy
3. Run Claude Code pointing to the LiteLLM proxy port (usually 4000):
export ANTHROPIC_BASE_URL="http://localhost:4000"
export ANTHROPIC_API_KEY="dummy"
claude

2. ClawCode (Harness CLI)

Configure it via environment variables before launching your workspace:
export OPENAI_BASE_URL="http://localhost:8787/v1"
export OPENAI_API_KEY="dummy"
clawcode --model auto

3. Cursor IDE

1. Open Cursor Settings > Models.
2. Add a new OpenAI-compatible Base URL.
Base URL: http://localhost:8787/v1
API Key: dummy
3. Add Custom Model Names (e.g., auto, claude-3.5-sonnet).
4. Toggle them on to use them in chat/composer.

4. OpenClaw (Autonomous OS Agent)

Configure the custom provider in your `~/.openclaw/openclaw.json`:
"models": {
  "providers": {
    "proxipt": {
      "base_url": "http://localhost:8787/v1",
      "api": "openai-completions",
      "api_key": "dummy"
    }
  }
}

5. Zed IDE

Open your settings.json via cmd+, and configure:
"language_models": {
  "openai": {
    "api_url": "http://localhost:8787/v1",
    "available_models": [{"name": "auto", "max_tokens": 128000}]
  }
}

6. AutoGPT

Route the Python agent loop through your local endpoint in `.env`:
OPENAI_API_BASE_URL="http://localhost:8787/v1"
OPENAI_API_KEY="dummy"
SMART_LLM="auto"
FAST_LLM="auto"

Session Extraction Guide

클라우드 환경(GCP 등)에서 세션 유지를 위해 필요한 session.json 파일을 획득하는 방법입니다.

방법 1: 로컬 proXiPT 사용 (권장)

가장 정확하고 오류가 없는 방법입니다. Playwright가 요구하는 모든 데이터(Cookies + LocalStorage)를 자동으로 캡처합니다.

  1. 사용 중인 로컬 PC에서 proXiPT를 실행합니다.
  2. 대시보드(Overview)에서 원하는 프로바이더의 Login (GUI) 버튼을 눌러 로그인합니다.
  3. 로그인이 완료되면 GUI를 닫습니다.
  4. proXiPT 설치 폴더의 sessions/ 디렉토리에 [provider]_session.json 파일이 생성됩니다.
  5. 이 파일을 클라우드 대시보드에서 Upload JSON 버튼을 통해 업로드합니다.

방법 2: 브라우저 확장 프로그램 사용 (수동)

proXiPT를 로컬에서 실행하기 번거로운 경우, 브라우저에서 직접 쿠키를 JSON으로 내보낼 수 있습니다.

1. 확장 프로그램 설치:
  • Chrome / Edge / Brave: Cookie-Editor 추천
  • Firefox: Cookie-Editor 추천

2. 세션 데이터 추출:
  1. 해당 프로바이더 사이트(예: chatgpt.com)에 접속하여 로그인합니다.
  2. 확장 프로그램을 실행하고 Export -> JSON을 클릭하여 클립보드에 복사합니다.
  3. 텍스트 편집기를 열어 아래 형식을 만들고, [복사한 내용] 부분에 붙여넣습니다:
    { "cookies": [복사한 내용], "origins": [] }
  4. 파일을 session.json으로 저장한 뒤 대시보드에 업로드합니다.

* 주의: 일부 프로바이더는 LocalStorage(origins) 데이터가 없으면 정상 작동하지 않을 수 있습니다.

방법 3: 개발자 도구 활용 (브라우저별 위치)

별도의 설치 없이 브라우저 내장 도구를 사용하여 세션 값을 확인할 수 있습니다.

Chrome / Brave / Edge
  • 단축키: F12 또는 Ctrl+Shift+I
  • 탭: Application (애플리케이션)
  • 좌측 메뉴: Storage -> Cookies
Firefox
  • 단축키: F12 또는 Ctrl+Shift+I
  • 탭: Storage (저장소)
  • 좌측 메뉴: Cookies (쿠키)
